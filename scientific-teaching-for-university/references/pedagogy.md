# 教学法参考（Mode 3 / Mode 4 深读）

这份文件展开《核心教学理念》，重点回答一个问题：**面对一个对初学者特别难 / 特别抽象 / 数学类的概念，具体该怎么一步一步讲？** Mode 3 和 Mode 4 讲难概念时以此为准。

## 一、初学者视角是第一原则

最常见、也最致命的错误：用"已经懂了的人"的上帝视角讲课。专家看一个概念，看到的是它在整张知识网里的位置；初学者看到的只是一堆陌生符号和术语。两者的认知起点完全不同。

具体要求：
- 不要假设用户"应该已经知道"某个前置概念。如果一个解释依赖某个前置知识，先花一两句把前置补上，或至少点明"这里用到了 X，它大致是……"。
- 不要跳步。专家眼里"显然"的一步，对初学者往往正是卡住的地方。
- 用词从"人话"起步，术语后置。先让人理解意思，再告诉他"这个东西的正式名字叫……"。

## 二、讲一个难概念的推荐顺序

按下面的顺序走，能显著降低理解门槛：

1. **定位（Why & Where）**：这个概念在整门课/这个领域的哪个位置？为什么这里需要它？它想解决什么问题？——先给坐标，再讲内容。

2. **直觉（Intuition first）**：用一个简单、贴近生活或已知经验的类比 / 图景，先让人"感觉到"这个概念大概是什么。intuition 要简单但**不生硬**——不能为了简单而扭曲本意，也不能是套话。好的 intuition 让人"哦～原来是这个意思"。

3. **最小例子（Minimal example）**：给一个尽量小、尽量具体的例子，把抽象概念落到看得见摸得着的实例上。数字越小越好，步骤越少越好，先让人在具体上跑通一遍。

4. **正式定义（Formalize）**：这时才引入严谨的、形式化的定义和符号。因为前面铺垫过，此刻的符号不再是天书，而是"给刚才那个直觉起个正式名字"。

5. **评价（Evaluation of concept）**：这个概念的意义、适用范围、局限、和相邻概念的区别与联系。帮学习者建立"什么时候用它、什么时候不用"的判断力。

6. **再来一两个 example**：可以是课内的（对齐考试），也可以是课外的（拓宽理解），巩固。

## 三、讲数学公式的方法

数学是初学者最容易被"符号劝退"的地方。讲公式时：

1. **先讲这个公式"想干什么"**：抛开符号，先用一句人话说清它的目的/直觉。例如"这个式子其实就是在量：预测值和真实值差了多少"。
2. **逐符号解释**：把公式里每个符号 / 下标 / 求和号是什么意思，一个个讲清楚。不要默认用户认识 \( \sum \)、\( \nabla \)、\( \mathbb{E} \) 这类记号。
3. **把符号"读"成一句话**：把整个公式用中文读一遍，像读句子一样。
4. **代一个最小的具体数字**：用极小的例子把公式实际算一遍，让抽象的式子在具体数字上"活"起来。
5. **最后当作整体使用**：这时再把它作为一个完整工具来用于后续推导。

排版：行内公式用 \( ... \)，独立公式用 \[ ... \]。

## 四、Mode 4：当讲完还是不懂时怎么办

Mode 4 的场景是：Mode 3 已经讲过某个难概念，但用户的基础还没到能直接理解的程度。这时**不要重复原来的讲法**（原来的讲法已经被验证太难了），而要**降维重来**：

- **先共情、先定位卡点**：承认之前讲得对当前基础来说太快/太抽象。如果能判断，指出用户可能卡在哪个前置环节。
- **把台阶拆得更细**：从更基础的地方重新起步，把原来"一步"拆成"三步五步"，一步一步向上推，直到推到那个复杂理解为止。
- **换一种讲法**：如果原来的类比没奏效，换一个全新的类比 / 图景 / 例子，而不是把同样的话再说一遍。
- **每个新符号、新术语都先解释再用**：绝不跳步，绝不出现"你应该已经知道"。
- **多用具体、少用抽象**：能用数字就用数字，能画图景就描述图景，把抽象概念反复落到具体上。

目标只有一个：让基础不足的人也能真正跟上并理解这个概念。

## 五、自检清单

讲完一页 / 一个概念后，快速自检：
- [ ] 我有没有从初学者视角出发，而不是上帝视角？
- [ ] 难概念是否给了简单但不生硬的 intuition？
- [ ] 是否给了 example（课内/课外）？
- [ ] 是否说明了它与上下文的关系、以及老师为什么放在这里？
- [ ] 术语是否从"人话"起步、逐步引入英文术语？
- [ ] 公式是否逐符号讲清、并代过最小具体例子？
