# Teaching-Method Reference (deep read for Mode 3 / Mode 4)

This file expands on the Core Teaching Philosophy, focusing on one question: **when facing a concept that is especially hard / especially abstract / mathematical for a beginner, exactly how should you teach it step by step?** Use this as the standard when Mode 3 and Mode 4 explain hard concepts.

## 1. The beginner's perspective is the first principle

The most common — and most fatal — mistake: teaching from the god's-eye view of "someone who already understands." An expert looking at a concept sees its position in the whole web of knowledge; a beginner sees only a pile of unfamiliar symbols and terms. Their cognitive starting points are completely different.

Concrete requirements:
- Don't assume the user "should already know" some prerequisite concept. If an explanation depends on a prerequisite, spend a sentence or two filling it in first, or at least point out "here we use X, which roughly means…".
- Don't skip steps. The step that is "obvious" to an expert is often exactly where the beginner gets stuck.
- Start with plain language and defer the terminology. Let the person understand the meaning first, then tell them "the formal name for this thing is…".

## 2. Recommended order for teaching a hard concept

Following the order below significantly lowers the barrier to understanding:

1. **Locate (Why & Where):** where does this concept sit in the whole course / field? Why is it needed here? What problem is it trying to solve? — Give the coordinates first, then the content.

2. **Intuition first:** use a simple analogy / mental picture close to everyday life or known experience, so the person first "feels" roughly what this concept is. The intuition should be simple but **not forced** — it must not distort the true meaning for the sake of simplicity, nor be an empty cliché. A good intuition makes the person go "oh, so that's what it means."

3. **Minimal example:** give an example that is as small and concrete as possible, grounding the abstract concept in a tangible instance. The smaller the numbers the better, the fewer the steps the better; let the person run through it once on something concrete first.

4. **Formalize:** only now introduce the rigorous, formal definition and notation. Because you've laid the groundwork, the symbols are no longer gibberish but "giving a formal name to the intuition we just had."

5. **Evaluation of the concept:** the concept's significance, scope of application, limitations, and its differences from and connections to neighboring concepts. Help the learner build the judgment of "when to use it and when not to."

6. **One or two more examples:** these can be in-course (aligned with exams) or out-of-course (broadening understanding), to consolidate.

## 3. How to teach a mathematical formula

Math is where beginners are most easily "scared off by symbols." When teaching a formula:

1. **First say what the formula is "trying to do":** set the symbols aside and state its purpose/intuition in one plain sentence. For example, "this expression is really just measuring how far the prediction is from the true value."
2. **Explain symbol by symbol:** clarify what every symbol / subscript / summation sign means, one at a time. Don't assume the user knows notation like \( \sum \), \( \nabla \), \( \mathbb{E} \).
3. **"Read" the symbols as a sentence:** read the whole formula in plain language, like reading a sentence.
4. **Plug in a minimal concrete number:** actually compute the formula once with a tiny example, so the abstract expression comes "alive" on concrete numbers.
5. **Finally use it as a whole:** only now treat it as a complete tool for subsequent derivations.

Typesetting: inline math with \( ... \), display math with \[ ... \].

## 4. Mode 4: what to do when it's still unclear after explaining

Mode 4's scenario: Mode 3 has already explained a hard concept, but the user's foundation hasn't reached the point of understanding it directly. Here, **don't repeat the original explanation** (it's already been proven too hard) — instead, **scale down and start over**:

- **Empathize first, then locate the sticking point:** acknowledge that the previous explanation was too fast/abstract for the current level. If you can tell, point out which prerequisite step the user might be stuck on.
- **Break the steps into finer pieces:** restart from a more basic level, splitting the original "one step" into "three or five steps," building up step by step until you reach that complex understanding.
- **Switch to a different explanation:** if the original analogy didn't land, use a completely new analogy / mental picture / example rather than repeating the same words.
- **Explain every new symbol and term before using it:** never skip steps, never say "you should already know this."
- **Use more concrete, less abstract:** use numbers where you can, describe a mental picture where you can, repeatedly grounding the abstract concept in the concrete.

There is only one goal: to let even someone with an insufficient foundation genuinely follow along and understand the concept.

## 5. Self-check list

After finishing a page / a concept, do a quick self-check:
- [ ] Did I start from the beginner's perspective, not the god's-eye view?
- [ ] Did the hard concept get a simple but not-forced intuition?
- [ ] Did I give an example (in-course/out-of-course)?
- [ ] Did I explain its relationship to the context, and why the instructor placed it here?
- [ ] Did the terminology start from plain language and gradually introduce the formal terms?
- [ ] Was the formula explained symbol by symbol, with a minimal concrete example computed?
